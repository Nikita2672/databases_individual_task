# Database Infrastructure

Эта папка относится только к индивидуальному заданию по базам данных и ведется как отдельный git-репозиторий.

## Structure

- `postgresql` — Docker Compose и init-скрипты для PostgreSQL
- `mongodb` — Docker Compose и init-скрипты для MongoDB

## Task

- В настоящее время я занимаюсь исследованиями в области коллаборативных приложений, в частности — формализацией и разработкой подходов к представлению композитных объектов на основе CRDT-структур данных. В качестве прототипа реализуется календарное приложение с поддержкой оффлайн-режима и последующей синхронизацией, на котором апробируются мои гипотезы касаемо модели конфликтов в распределенных системах.

- Понимая, что сама тематика CRDT относится преимущественно к распределённым системам, предлагаю связать её с содержанием курса следующим образом, сделав индивидуальное задание по следующей теме:

- Проектирование и анализ схемы хранения композитных объектов, представляемых на основе pure operation-based CRDT, в системе управления базами данных.

- В рамках работы предполагается: – разработка схемы хранения операций, а также векторных часов; – анализ нормализации и целостности данных; – проектирование индексов и анализ типовых запросов; – сравнение возможных вариантов хранения (реляционная модель, например PostgreSQL, и документная модель, например MongoDB). На текущем этапе исследования окончательный выбор типа СУБД (реляционная или документная) ещё не зафиксирован, поскольку рассматривается гипотеза о возможных преимуществах хранения операций в JSON-представлении. В рамках задания можно провести обоснование выбранной модели хранения. Буду признателен за Ваше мнение и возможные корректировки формулировки задания.

## PostgreSQL

Запуск:

```bash
docker compose -f infra/postgresql/compose.yaml up -d
```

Остановка:

```bash
docker compose -f infra/postgresql/compose.yaml down
```

Параметры подключения:

- host: `localhost`
- port: `55432`
- database: `crdt_lab`
- user: `crdt`
- password: `crdt`

Основные сущности:

- `replicas`
- `objects`
- `operations`
- `operation_dependencies`
- `vector_clock_entries`
- `object_snapshots`

## MongoDB

Запуск:

```bash
docker compose -f infra/mongodb/compose.yaml up -d
```

Остановка:

```bash
docker compose -f infra/mongodb/compose.yaml down
```

Параметры подключения:

- host: `localhost`
- port: `57017`
- database: `crdt_lab`
- user: `root`
- password: `root`

Коллекции:

- `replicas`
- `objects`
- `operations`
- `snapshots`
